**RemnaNode Advanced Installer** — это профессиональный, полностью русифицированный интерактивный скрипт автоматического развертывания, сетевой оптимизации, маскировки и мониторинга ноды управления **Remnawave** (`remnanode`) на серверах Ubuntu и Debian.

Проект объединяет лучшие практики сетевого тюнинга высоких нагрузок (Highload), глубокой обфускации маскировочных сайтов (Selfsteal), автоматического администрирования фаервола и гибкого управления SSL-сертификатами (Cloudflare DNS-01, Standalone, Gcore).

---

## 🎨 Архитектурный дизайн проекта

Нода разворачивается в изолированном Docker-окружении, состоящем из двух контейнеров, общающихся по сверхбыстрым Unix-сокетам в оперативной памяти:

```
                  Входящий трафик (Порт 443 TCP/UDP)
                                 │
                                 ▼
                     ┌───────────────────────┐
                     │   Xray-Core (Host)    │ <─── [Сертификаты в /etc/xray/certs]
                     └───────────┬───────────┘
                                 │
                 (Не-VPN HTTP трафик / Fallback)
                                 │
                                 ▼ (Unix-сокет: /dev/shm/nginx.sock)
                     ┌───────────────────────┐
                     │   Nginx (Decoy Site)  │ <─── [Мутированный HTML / Selfsteal]
                     └───────────────────────┘
```

* **Xray на хост-сети**: Нода Xray слушает внешний порт `443` хоста. Вся обработка VLESS Reality и TLS-рукопожатий происходит на первом рубеже.
* **Unix-сокеты в Shared Memory (`/dev/shm`)**: Для пересылки не-VPN трафика (активного зондирования цензоров или обычных браузеров) используется сокет в оперативной памяти хоста. Это полностью исключает накладные расходы сетевого стека TCP/IP.
* **Nginx на втором рубеже**: Контейнер `remnawave-nginx` принимает перенаправленный трафик от Xray по Unix-сокету и отдает уникальный, глубоко мутированный сайт-заглушку.

## 🔥 Ключевые возможности

### 1. ⚙️ Системный Тюнинг и Оптимизация (BBR & Hardening)
Скрипт автоматически анализирует объем оперативной памяти сервера и применяет оптимизированные параметры ядра `sysctl` и системные лимиты `/etc/security/limits.d/`:
* **BBR**: Принудительно активирует алгоритм контроля сетевых перегрузок TCP BBR от Google и планировщик пакетов `fq` для минимизации потерь пакетов и ускорения передачи данных.
* **Профиль HIGHLOAD** (для VPS с ОЗУ $\ge 2$ ГБ): Разгоняет лимиты дескрипторов файлов `nofile` до `1 048 576` и емкость таблицы отслеживания сетевых соединений `nf_conntrack` до `1 048 576` для работы под экстремальными нагрузками.
* **Профиль SAFE** (для VPS с ОЗУ $< 2$ ГБ): Бережные, экономные лимиты во избежание сбоев из-за нехватки оперативной памяти.

### 2. ⚡ Версионный менеджмент Xray-Core
Скрипт опрашивает официальный GitHub API репозитория `XTLS/Xray-core` и выводит интерактивный список последних 10 релизов:
* Позволяет установить любую версию ядра Xray (включая **Pre-release** сборки).
* Поддерживает «горячую» замену версии ядра на лету через перезапуск контейнера.
* Автоматически определяет архитектуру процессора (`amd64` / `arm64`) для скачивания нужного бинарного файла.

### 3. 🎭 Уникальный мутатор маскировки Selfsteal (Анти-фингерпринтинг)
Для борьбы с системами активного зондирования DPI (такими как ТСПУ РФ или GFW Китая) скрипт производит глубокую рандомизацию шаблонов сайтов-заглушек:
* **Генерация идентичности**: Создает реалистичные случайные бренды компаний (например, *"Lumen Cloud"*, *"Apex Studio"*) и встраивает их в теги `<title>` и тело сайта.
* **Удаление следов**: Стирает мета-файлы, README, LICENSE, исходные карты кода (.map), выдающие происхождение шаблона.
* **Защита от слежки**: Вырезает внешние веб-маяки, трекеры IP (такие как `api.ipify.org`) и внешние шрифты Google Fonts во избежание утечек метаданных.
* **Цветовая мутация**: Случайным образом перестраивает цветовую гамму шаблона (hue-shift), делая цифровой отпечаток сайта уникальным для каждого сервера.

### 4. 🔌 Управление портами и автоматический UFW (Защита от локаута)
* **Анти-локаут**: Скрипт сканирует запущенные сокеты и автоматически определяет ваш текущий порт SSH (даже нестандартный). Он вносит разрешающее правило UFW перед включением фаервола, гарантируя, что администратор **не потеряет доступ** к серверу.
* **Привязка панели**: Входящий порт управления нодой (по умолчанию `2222`) закрывается для всего мира в UFW и открывается **строго для IP-адреса вашей панели Remnawave**.
* **Интерактивный менеджер портов**: Встроенное меню позволяет в один клик открывать и закрывать дополнительные клиентские TCP/UDP-порты Xray в фаерволе, автоматически синхронизируя базу портов.

### 5. 🔐 Множественные варианты выпуска SSL-сертификатов
Скрипт поддерживает три метода прохождения валидации Let's Encrypt для VLESS+TLS:
1. **Cloudflare DNS-01 API**: Выпуск скрытого Wildcard-сертификата (`*.домен`), скрывающего структуру ваших поддоменов от систем логирования Certificate Transparency. Поддерживает авторизацию как по токенам (API Token), так и по глобальным ключам.
2. **Certbot Standalone (HTTP-01)**: Классический метод с автоматическим временным открытием порта 80 в UFW.
3. **Gcore DNS-01 API**: Автоматическая установка плагина `certbot-dns-gcore` через pip и выпуск Wildcard-сертификатов.
* **Renewal Hook**: Настроен автоматический крон-скрипт обновления сертификатов Let's Encrypt с их автокопированием в защищенную папку ноды и перезапуском контейнера.

### 6. 📊 Продвинутая диагностика и вынос логов
* Логи Xray (`access.log` и `error.log`) и Nginx выносятся на хост в папку `/var/log/remnanode/`.
* Настроена автоматическая ежедневная ротация `logrotate` при достижении размера файла **50 МБ** с хранением до 7 архивных копий.
* Интерактивное подменю диагностики выводит загрузку CPU, ОЗУ, диска хоста, статусы докеров и размеры всех файлов логов в реальном времени.

## 🚀 Быстрый старт

Запустите скрипт одной командой на вашем чистом сервере Ubuntu/Debian:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Balbuto/setup-remna-node/main/setup_node.sh)
```

> **Важно**: При первом запуске скрипт автоматически зарегистрирует себя в глобальной CLI-оболочке. В дальнейшем вы сможете вызывать интерактивное меню из любой папки терминала простой командой:
> ```bash
> remnanode
> ```

## ⚙️ Требования к серверу
* **Операционная система**: Ubuntu 20.04 / 22.04 / 24.04 LTS или Debian 11 / 12.
* **Права**: Доступ с правами `root`.
* **Домен**: Зарегистрированный домен (A-запись должна указывать на IP вашего сервера ноды).
