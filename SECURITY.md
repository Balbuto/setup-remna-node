## 🕵️ Модель угроз ноды VPN

Нода VPN (`remnanode`), развернутая на публичном IP-адресе, подвержена следующим типам сетевых атак и сканирования:

1. **Активное зондирование (Active Probing)**: Системы автоматической цензуры (DPI) совершают TLS/HTTP запросы на порт `443` вашего сервера. Если порт отвечает ошибкой TLS или нетипичным поведением Xray, сервер классифицируется как прокси и блокируется.
2. **Сканирование портов управления (Port Scanning)**: Злоумышленники сканируют порт управления Collie API (по умолчанию `2222`), пытаясь перехватить контроль над нодой или совершить brute-force атаку.
3. **Утечки метаданных и отпечатков пальцев (Fingerprinting)**: Веб-серверы маскировки, отдающие стандартные неизмененные HTML-шаблоны или использующие внешние шрифты Google, легко детектируются цензорами по идентичности контента (hash-суммы файлов, структура CSS-стилей).

## 🛡️ Реализованные механизмы защиты

Скрипт по умолчанию внедряет комплексную систему безопасности операционной системы и контейнеров:

### 1. Сетевая изоляция порта управления (`NODE_PORT`)
Входящий порт управления `NODE_PORT` (по умолчанию `2222`), через который панель Remnawave отправляет команды ноде, жестко блокируется в брандмауэре **UFW** для всего мира. Доступ к нему разрешается **исключительно для IP-адреса вашей управляющей панели**:
```bash
ufw allow from "$panel_ip" to any port "$node_port" proto tcp
```
Это полностью исключает возможность сканирования, brute-force атак и эксплуатации потенциальных уязвимостей API Collie из внешней сети.

### 2. Защита от локаута по SSH
При активации брандмауэра UFW неопытные администраторы часто забывают открыть порт SSH, что приводит к полной потере контроля над сервером. Наш скрипт осуществляет **автоопределение активного порта SSH** на хост-системе (путем парсинга активных сокетов `sshd`) и автоматически добавляет разрешающее правило UFW перед активацией фаервола.

### 3. Механизм маскировки (Selfsteal) и мутации (Анти-фингерпринтинг)
Контейнер `remnawave-nginx` выступает в качестве легитимного прикрытия для сетевого трафика. Мутатор маскировочных сайтов защищает ноду от активного зондирования:
* **Мутация бренда**: В тело сайта внедряются уникальные бренды, мета-теги идентификации и случайные HTML-комментарии.
* **Локализация ресурсов**: Удаляются любые ссылки на внешние шрифты, трекеры и beacons, которые могут заблокировать ваш сервер или выдать его IP.
* **Сдвиг спектра цветов (CSS Hue Shift)**: Цвет элементов дизайна шаблона сдвигается на случайный градус, что делает код файлов стилей уникальным на каждом конкретном сервере.

### 4. Безопасность SSL и скрытие поддоменов (DNS-01 Wildcard)
* При использовании классического выпуска Let's Encrypt через Standalone-метод, порт `80` открывается в UFW **только на время прохождения валидации**, после чего автоматически закрывается.
* При использовании выпуска через API **Cloudflare** или **Gcore** (протокол DNS-01) выпускается **Wildcard-сертификат** формата `*.вашдомен.com`. Это скрывает имя поддомена вашей ноды (например, `node-us-05.domain.com`) из публичных логов Certificate Transparency.

## 🔒 Рекомендации по безопасной эксплуатации

1. **Используйте API-токены Cloudflare вместо Global API Key**: При выпуске Wildcard-сертификата используйте ограниченные API-токены с правами на редактирование только конкретной DNS-зоны. Это защитит ваш Cloudflare-аккаунт в случае компрометации сервера.
2. **Регулярно обновляйте Xray-Core**: С помощью пункта `3` интерактивного меню своевременно обновляйте ядро Xray на стабильные версии для защиты от известных уязвимостей протоколов.
3. **Отключайте IPv6 при отсутствии явной необходимости**: Пункт `5` меню позволяет выключить IPv6 на сетевом уровне ядра. Это предотвратит возможные утечки реального IP-адреса хоста через туннели IPv6 (IPv6 Leakage).
4. **Не открывайте неиспользуемые порты**: Пользуйтесь пунктом `8` меню для точечного управления правилами UFW. Держите открытыми только те порты, которые фактически прописаны в панели управления для входов ваших клиентов.
