# 📝 История изменений (CHANGELOG.md)

Все заметные изменения в проекте **RemnaNode Advanced Installer & Optimizer** документируются в этом файле.

Формат ведения истории основан на принципах [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/), а версии соответствуют семантическому версионированию [Semantic Versioning](https://semver.org/lang/ru/).

---

## [1.7.1] — 2026-06-23

### Добавлено
* **Принудительная проверка root на первой строчке**: Проверка прав суперпользователя перенесена в самое начало выполнения скрипта, до инициализации лог-файлов. Это полностью предотвращает ошибку `Permission Denied` и аварийное падение Bash при случайном запуске обычным пользователем без `sudo`.
* **Доработка удаленной регистрации и установки**: В README.md обновлены строки однострочной удаленной установки. Запуск через process substitution `<(curl ...)` (который часто падал на ошибку `/dev/fd/63: No such file or directory` из-за конфликта прав `sudo` и виртуальных дескрипторов) заменен на 100% стабильный и совместимый pipe-формат: `curl -fsSL ... | sudo bash`.

### Исправлено
* **Подкоманда Ping/Traceroute в мульти-тестах**: Исправлено отображение маршрутов до yandex.ru, добавлена автоматическая установка недостающего пакета `traceroute` в процессе выполнения тестов.

## [1.7.0] — 2026-06-23

### Добавлено
* **Раздел мульти-тестирования сервера (Пункт 10)**: Интегрирован мощный модуль сетевой и аппаратной диагностики хоста на базе наработок `Balbuto`.
* **10 встроенных сценариев тестирования**:
  - `YABS` — тест дисков (fio), процессора (Geekbench) и сети.
  - `Censorcheck` — проверка провайдера на геоблоки и DPI-инспекцию.
  - `iPerf3 RU Speedtest` — скорость до РФ-узлов.
  - `IP Check Place` — репутация и Fraud Score вашего IP-адреса.
  - `sysbench CPU & Memory` — аппаратный стресс-тест ядер и ОЗУ.
  - `Ping & Traceroute` — проверка задержек и маршрутов до yandex.ru.
  - Режим `Запустить ВСЕ тесты по очереди` для полной диагностики.
* **Безопасный загрузчик `run_verified_benchmark`**: Все тесты скачиваются строго по зашифрованному протоколу **TLS 1.2+** и выполняются в изолированных подпроцессах (subshell). Падение или сбой теста не приводит к падению главного меню установщика.

### Исправлено
* **Исправление багов логов Xray (v1.6.5)**: Локализован баг пустых логов на хосте. Пути томов в Docker Compose перенаправлены на внутреннюю Supervisord-папку контейнера (`/var/log/supervisor`). Логи `xray.out.log` и `xray.err.log` теперь транслируются на хост в реальном времени.
* **Баг удаленного запуска CLI (v1.6.6)**: Исправлена ошибка `command not found` при первом запуске через `curl`/`wget`. Внедрен интеллектуальный детектор удаленной среды (`Remote Pipe Detection`). При запуске в памяти скрипт автоматически скачивает свою стабильную физическую копию по `SCRIPT_URL` в `/usr/local/bin/remnanode`.
* **Безопасный недеструктивный фаевол (v1.6.3)**: Из функции `configure_firewall` полностью удалена деструктивная системная команда `ufw reset`, стиравшая правила других служб хоста. Внедрена точечная замена IP панели: при переезде панели старое UFW-правило для старого IP удаляется, а новое для нового IP записывается. Все правила SSH/HTTP/HTTPS/кастомные порты остаются нетронутыми.

## [1.6.1] — 2026-06-23

### Добавлено
* **Поддержка протокола VLESS+TLS+xHTTP**: Добавлен новый высокопроизводительный транспортный протокол, устойчивый к блокировкам. Настроена бесшовная автоматическая прописка и копирование SSL-сертификатов Let's Encrypt в контейнер ноды.
* **Интерактивное меню смены шаблонов маскировки (Пункт 9)**: Возможность изменять дизайн сайта-прикрытия в реальном времени. Реализованы два сценария: смена на **Случайный (Рандомно)** или выбор **Конкретного** из 8 профессиональных адаптивных макетов.
* **Автоматическая чистка и перезапуск**: После смены темы контейнер Nginx перезапускается автоматически для применения изменений.

## [1.6.0] — 2026-06-22

### Добавлено
* **Интеграция с Mrvibecodic/node-templates**: В проект встроена коллекция из 8 профессиональных адаптивных шаблонов заглушек (`playza-game-catalog`, `levelup-hub`, `rybaliti-2.0` и др.).
* **Интеграция мутатора `uniquify-theme.sh`**: Добавлена контекстно-безопасная мутация кода "на лету" на базе Perl.
* **Анти-фингерпринтинг**: Реализовано автоматическое шифрование DOM-структуры (переименование классов CSS, ID и CSS-переменных), случайная инъекция комментариев и Whitespace Jitter (джиттер пробелов) для уникализации хэшей и размеров файлов.
* **Унификация Nginx**: Контейнер `remnawave-nginx` теперь устанавливается и работает **всегда** (для TLS, xHTTP и Reality режимов), гарантируя наличие легитимного fallback-сайта.

### Исправлено
* **Синтаксис `sed`**: Исправлена синтаксическая ошибка `sed: unknown option to 's'` при вырезании Google-шрифтов. Символы разделителей изменены на безопасные `#`, замена переведена на чистую пустую строку.
